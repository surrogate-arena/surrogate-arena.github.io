🎯 Goal

Find the Pareto front of competing objectives as quickly as possible — maximizing hypervolume (the volume dominated by found solutions) over BO iterations.

Every surrogate uses the same acquisition function and optimizer, so differences in the HV curves are purely due to surrogate quality.

⚙️ Experimental Setup

  • Acquisition function: qLogEHVI (q-batch Log Expected Hypervolume Improvement)
  • Optimizer: Simulated Annealing over a finite candidate pool
  • Batch size: q = 3 points per iteration
  • Iterations: 50 per run
  • Seeds: 5 random seeds (different init data)
  • Hardware: NVIDIA GB10 GPU (DGX1)

🧩 The Lego Design

Surrogate Arena is built as three interchangeable blocks:

  • Surrogate — models the objective function(s) with uncertainty
  • Acquisition function — decides where to sample next using the surrogate's predictions
  • Optimizer — maximizes the acquisition function to pick the actual candidate

📊 Reading the Charts

  • X-axis: BO iteration (0 = initial random data)
  • Y-axis: Hypervolume dominated by the current Pareto set
  • Higher is better — larger hypervolume means better coverage of the Pareto front
  • Shaded bands: ±1 standard deviation across 5 seeds
  • Narrow band = consistent across seeds (low variance)

🤖Surrogate Models

All surrogates are wrapped as BoTorch Model objects exposing a .posterior(X) interface. Each is plugged into the same qLogEHVI acquisition function.

📐Benchmark Functions

Standard multi-objective test functions used to evaluate BO methods. Each is a deterministic black-box function evaluated at the candidates proposed by the BO loop.

🏆 Why Hypervolume?

Hypervolume (HV) is the volume of the objective space dominated by the current set of Pareto-optimal solutions relative to a fixed reference point. It is a single scalar that captures both the quality and diversity of the Pareto front. Higher HV = better coverage of the true Pareto front. It is the gold-standard metric for multi-objective optimization.
